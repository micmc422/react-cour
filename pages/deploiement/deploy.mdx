# Déploiement d'une application React sur Vercel ou Netlify

## Déploiement sur Vercel

Vercel est une plateforme pour le déploiement rapide des applications React. Elle offre une intégration continue instantanée avec GitHub, GitLab ou Bitbucket, ce qui signifie que chaque fois que vous poussez des modifications sur votre dépôt, Vercel déploiera automatiquement une nouvelle version de votre application [Source 0](https://vercel.com/guides/deploying-react-with-vercel).

Pour déployer votre application React sur Vercel, suivez les étapes suivantes :

1. Créez un compte sur Vercel.
2. Installez le CLI de Vercel en exécutant `npm i -g vercel` dans votre terminal.
3. Dans le répertoire racine de votre projet, exécutez la commande `vercel` pour initialiser le déploiement.
4. Suivez les instructions à l'écran pour connecter votre projet à Vercel et configurer le déploiement.

Si vous utilisez GitHub, GitLab ou Bitbucket, vous pouvez également configurer le déploiement continu en connectant votre dépôt à Vercel. Chaque fois que vous poussez des modifications sur votre dépôt, Vercel déploiera automatiquement une nouvelle version de votre application [Source 1](https://blog.logrocket.com/deploy-react-app-for-free-using-vercel/).

## Déploiement sur Netlify

Netlify est une autre plateforme populaire pour le déploiement d'applications React. Tout comme Vercel, Netlify offre une intégration continue instantanée avec GitHub, GitLab et Bitbucket.

Pour déployer votre application React sur Netlify, suivez les étapes suivantes :

1. Créez un compte sur Netlify.
2. Dans le tableau de bord de Netlify, cliquez sur "New site from Git".
3. Connectez votre dépôt GitHub, GitLab ou Bitbucket à Netlify.
4. Configurez les paramètres de build et de deploy de votre application.
5. Cliquez sur "Deploy site" pour déployer votre application.

Comme Vercel, Netlify déploiera automatiquement une nouvelle version de votre application chaque fois que vous pousserez des modifications sur votre dépôt.

En conclusion, le choix entre Vercel et Netlify dépend principalement de vos préférences personnelles et des besoins spécifiques de votre projet. Les deux plateformes offrent une intégration continue instantanée, une gestion facile du déploiement et une excellente performance.
